# Бухгалтерия

## Назначение

Чат **«Бухгалтерия»** — отдельная Telegram-группа куда оператор кидает отчёты по обменным заявкам в формате СТАРТ. Юзербот парсит их, считает маржу и **АВТОМАТОМ ПЕРЕВОДИТ ВСЕ ЛК ЗАЯВКИ В `ГОТОВО_К_ОТПУСКУ`**.

ID хранится в `storage.accounting_group_id`. Задаётся через `/admin → 🧠 AI → 📊 Бухгалтерия чат ID`.

## Связь между чатами (важно!)

```
Чат «Бухгалтерия» (СТАРТ-отчёт)
        │
        ├──► storage.accounting (запись заявки)
        │
        └──► для каждого output ЛК:
              storage.find_deal_by(fio, bank)
              ├──► _apply_status_change(deal, "ГОТОВО_К_ОТПУСКУ")
              │     ├──► пост в чате «Сделки и выплаты» обновляется
              │     ├──► пост в чате «Отработка аккаунтов» обновляется
              │     │     (статус: УСПЕШНО ОТРАБОТАНО)
              │     └──► клиенту в work_chat: «сделка готова к отпуску»
              └──► _send_release_request
                    └──► в чат «Сделки и выплаты»:
                         «❓ Сделка #ID отработана — отпустить
                          в гаранте? @TimonSkupCL»
                         (или USDT-выплата по deal.method)
```

**Один отчёт оператора в бухгалтерию = всё остальное автоматом.**

## Формат СТАРТ (как пишет оператор)

```
СТАРТ 05.05:
1. Заявка 785000 - откуп 771610
ПРИЕМ: АЛЬФА - Бреев Никита Александрович (остаток 1420р) - ЦЕНА ЛК 400$
Вывод:
ОЗОН Столярова - 227к - блок 1.5к - ЦЕНА ЛК 400$
ОЗОН Богорубов - 161к - остаток 3к дебет - ЦЕНА ЛК 400$
ТОЧКА Зоткин - 229к - блок 4к - ЦЕНА ЛК 400$
ОЗОН Муратшина - 165к - ЦЕНА ЛК 400$
🟢 Выплата клиенту: (785000-40%)/82 = 5743$
✅ Выплата нам (курс 82-ставка 2%) = 9.221$
```

### Что значит каждая строка

| Часть | Смысл |
|---|---|
| **Заявка** | Сумма в рублях которую партнёр-клиент переводит нам на обмен в USDT |
| **Откуп** | Сколько мы фактически получили после партнёрской комиссии (в рублях) |
| **ПРИЕМ** | Банк/ФИО **приёмного** ЛК (где партнёр шлёт нам деньги). Это рабочий счёт партнёра — в маржу не включается |
| **Вывод** | Список выводных ЛК (наши купленные счета поставщиков), через которые мы выводим в крипту |
| **Выплата клиенту** | Что отдаём партнёру после нашей комиссии (в USDT) |
| **Выплата нам** | Сколько мы получили (откуп / курс − наша ставка %) |
| **ЦЕНА ЛК** | Сколько мы заплатили клиенту-поставщику за каждый ЛК (если оператор не указал — берётся из storage.deals + прайса) |

### Расчёт маржи

```
our_usdt = (buy_amount_rub / course) * (1 - our_rate_pct/100)
client_usdt = (partner_amount_rub * (1 - client_pct/100)) / course
margin = our_usdt - client_usdt - сумма(ЛК выводов)
```

Приёмный ЛК (АЛЬФА в примере) **не учитывается** в марже — это рабочий счёт партнёра, не покупаем под каждую заявку.

### Зарплаты (в дневном отчёте)

- **Операционист**: 15% от суммы маржи всех заявок дня
- **Откупщик**: 2% от суммы маржи всех заявок дня
- **Чистая маржа компании**: маржа − зарплаты

## Auto-resolve цен ЛК

Если оператор не написал «ЦЕНА ЛК X$» в строке вывода — юзербот сам подставит цену по приоритету:

1. **Из текста оператора** (если указано — не трогаем)
2. **Из `storage.deals`** — `find_deal_by(fio=, bank=)` → `deal.amount`
3. **Из встроенного прайса** `accounting.PRICING_TABLE_USDT`:
   - Альфа 400$, ОЗОН 300$, Райф 350$, Точка 300$, Уралсиб 250$, ВТБ 300$
4. **0** если банк неизвестен

Прайс синхронизирован с `pricing.md`. Обновлять `PRICING_TABLE_USDT` в `accounting.py` ВРУЧНУЮ при изменении цен.

## Команды в чате «Бухгалтерия»

| Команда | Что делает |
|---|---|
| **СТАРТ-отчёт** (мульти-строка) | парсит, считает маржу, АВТО-обновляет все ЛК в ГОТОВО_К_ОТПУСКУ |
| `отчёт` | дневной отчёт за сегодня (заявки + правки + зарплаты) |
| `отчёт 2026-05-05` | за конкретную дату |
| `курс 95 100` | задать курс USDT (закуп / партнёру) |
| `приход 5000 название` | ручной приход |
| `расход 500 название` | ручной расход |
| `удали правку 2` | удалить ручную правку #N |
| `лк Альфа 400` | отдельная покупка ЛК (не из заявки) |
| `?` / `помощь` / `/help` | список команд |
