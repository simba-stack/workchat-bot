# Бухгалтерия V2 (Группа 2)

## Назначение

Группа 2 — это **операторская**. Сюда оператор подаёт `ЗАЯВКА` после каждой проводки. Бот:
- считает маржу с учётом блоков
- автоматически переводит все ЛК на выходе из `В_РАБОТЕ` в `ОТРАБОТАН`
- сохраняет заявку в `applications_v2` для отчётности

ID группы хранится в `accounting_group_id`.

---

## Формат заявки

```
ЗАЯВКА
Заход: 1500000 ₽
Курс заведения: 91.50
Курс пополнения: 92.00
Партнёр %: 5

Выходы:
- LK#A12 — Альфа — 400$
- LK#A15 — ОЗОН — 400$
- LK#A18 — Точка — 400$ — БЛОК 100000

Снято USDT: 16200
```

Поля:
- `Заход` — рублёвая сумма от клиента
- `Курс заведения` (course_w) — курс, по которому посчитан USDT нам
- `Курс пополнения` (course_p) — курс, по которому считаем выплату клиенту
- `Партнёр %` — процент партнёра (опционально, по умолчанию 0)
- `Выходы` — список карточек `LK#card_id` с банком и ценой; БЛОК и сумма указываются прямо в строке
- `Снято USDT` (total_withdrawn) — сколько USDT мы фактически забрали

---

## Формулы

```
we_got_usdt        = total_withdrawn / course_w
client_part_rub    = intake_rub * (1 - partner_pct/100)
client_payout_usdt = client_part_rub / course_p

для каждого выхода:
    if status == "БЛОК":
        block_usdt = block_rub / course_p
        if block_usdt >= price:
            effective = 0          # компенсация полная — ЛК выпадает
        else:
            effective = price - block_usdt
    else:
        effective = price

margin = we_got_usdt - client_payout_usdt - sum(effective for all outputs)
```

---

## Авто-каскад статусов

При успешной обработке `ЗАЯВКА`:
1. Все упомянутые ЛК (по `card_id`) → `ОТРАБОТАН`
2. Если у ЛК метод `GUARANTOR_AFTER_WORK` → бот тегает клиента в рабочем чате и просит создать сделку Conte → ждёт номер сделки → ставит `ПОПОЛНИТЬ_И_ОТПУСТИТЬ`
3. Если метод `USDT_TRC20` или `GUARANTOR_BEFORE_WORK` → ЛК остаётся `ОТРАБОТАН` до выплаты, после выплаты → `ЗАВЕРШЁН`

---

## Учёт блоков

В формуле блока **используется курс пополнения** (`course_p`), потому что блок — это рублёвая заморозка, которую мы должны вернуть клиенту по тому же курсу, по которому ему платим.

Пример: ЛК «Альфа 400$», блок 100 000 ₽, course_p = 91.50:
- block_usdt = 100000 / 91.50 ≈ 1093 USDT
- 1093 ≥ 400 → ЛК выпадает из маржи (effective = 0)

Пример: ЛК «Альфа 400$», блок 30 000 ₽, course_p = 91.50:
- block_usdt = 30000 / 91.50 ≈ 328 USDT
- 328 < 400 → effective = 400 − 328 = 72 USDT

---

## Сохранение

Заявка сохраняется в `applications_v2[YYYY-MM-DD][app_id]`. Хранит:
- intake, courses, partner_pct, total_withdrawn
- список выходов с финальными расчётами
- итоговую маржу
- timestamp

## Команды в Группе 2

Только парсинг `ЗАЯВКА`. Любое сообщение, начинающееся с этого ключевого слова, обрабатывается.
